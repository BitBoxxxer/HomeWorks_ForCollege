# Запуск проекта «Мониторинг температуры» (WPF + веб-панель)

### 1. Запустить сервер (консольное приложение C#)

- Открыть терминал (PowerShell, cmd или встроенный в VS Code).
- Перейти в папку с сервером:
  ```bash
  cd D:\GIT_HUB\__College\___Практика\28.04.26_Практика_Губин_Веб_WPF\TemperatureServer
  ```
- Запустить сервер:
  ```bash
  dotnet run --urls http://localhost:5000
  ```
- Дождаться сообщения:
  `Now listening on: http://localhost:5000`
  `Application started. Press Ctrl+C to shut down.`
- **Не закрывать этот терминал** – сервер должен работать всё время.

> Если порт 5000 занят, сервер сам выберет другой (например, 5001). Посмотрите на сообщение `Now listening on...` и используйте этот порт в следующих шагах.

### 2. Открыть веб‑панель (локальный сайт)

- Запустить браузер (Chrome, Edge, Firefox).
- Перейти по адресу:
  **`http://localhost:5000`** (или тот порт, который указан в терминале сервера).
- Увидите таблицу «Последние 10 показаний» – пока пустую.

### 3. Запустить WPF‑клиент (датчик)

- Открыть **новый** терминал (первый остаётся с сервером).
- Перейти в папку с WPF‑проектом:
  ```bash
  cd D:\GIT_HUB\__College\___Практика\28.04.26_Практика_Губин_Веб_WPF\WPF\UI
  ```
- Запустить WPF‑приложение:
  ```bash
  dotnet run
  ```
- Откроется окно «Датчик температуры». Нажать кнопку **«Старт»**.
- Каждые 5 секунд в окне будет появляться статус `✅ ... - OK`, а веб‑панель начнёт обновляться новыми данными.

### 4. Остановка проекта (когда закончили)

- **Остановить WPF‑клиент:** просто закрыть его окно (крестик).
- **Остановить сервер:** в терминале сервера нажать `Ctrl + C`.

## ⚠️ Возможные проблемы и их решение


| Проблема                                                               | Что делать                                                                                                                                                                                  |
| ------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Сервер не запускается, ошибка`address already in use` | Порт 5000 занят. Измените порт в команде:`dotnet run --urls http://localhost:5001` и во всех пунктах используйте `5001`.                       |
| Веб‑панель пустая (не показывает данные)     | Проверьте, что сервер запущен и WPF нажал «Старт». Обновите страницу в браузере (`F5`).                                               |
| WPF пишет`❌ Ошибка: Connection refused`                            | Сервер не запущен или запущен на другом порту. Убедитесь, что порты совпадают.                                                       |
| Файл`some_bd.json` не создаётся                                 | Если папка с проектами на диске`D:\` – проверьте права на запись. Можно временно указать полный путь в `Program.cs`. |
| Ошибка`NETSDK1080` при сборке                                   | Не страшно, это предупреждение. Игнорируйте.                                                                                                                    |

## 📝 Важные напоминания

- **Порядок важен:** сначала сервер, потом WPF, потом браузер.
- Сервер должен быть запущен **до** старта WPF и работать всё время.
- Можно запускать несколько экземпляров WPF (каждый будет отправлять температуру).
- Чтобы очистить историю, просто удалите `some_bd.json` или отредактируйте его.
